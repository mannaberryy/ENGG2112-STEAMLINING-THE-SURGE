Model 2: Sample-Level Prediction of COVID Deterioration

This model predicts future COVID deterioration from the metabolomic and clinical profile available at a single sample/presentation.

Multiple samples from the same patient are allowed as separate rows because each sample represents a separate clinical presentation. To prevent patient-level leakage, grouped cross-validation is used so all samples from the same patient stay in the same fold.

df["sample_day"] = df["day_from_sample"]

def make_sample_target(row):
    icu_time = row["time_from_symptom_onset_to_icu_days"]
    sample_day = row["sample_day"]
    death_30 = row["x30_day_mortality"]

    if pd.isna(sample_day):
        return np.nan

    if death_30 == 1:
        return "deteriorating"

    if pd.notna(icu_time) and icu_time > sample_day:
        return "deteriorating"

    if pd.notna(icu_time) and icu_time <= sample_day:
        return np.nan

    return "stable"

df["deterioration_binary"] = df.apply(make_sample_target, axis=1)

df_model = df.dropna(subset=["deterioration_binary", "patient_id"]).copy()

Cached Hyperparameter Tuning

# ============================================================
# CACHED GROUPED HYPERPARAMETER TUNING
# ============================================================

inner_cv = GroupKFold(n_splits=5)

tuned_results = {}
best_estimators = {}

for name, spec in model_specs.items():

    safe_name = name.replace(" ", "_").replace("/", "_")
    cache_path = os.path.join(CACHE_DIR, f"{cache_key}_{safe_name}_best_model.joblib")

    if os.path.exists(cache_path):
        print(f"\nLoading cached tuned model: {name}")

        cached = joblib.load(cache_path)

        best_estimators[name] = cached["best_estimator"]
        tuned_results[name] = cached["tuned_result"]

        continue

    print(f"\nTuning model: {name}")

    search = GridSearchCV(
        estimator=spec["pipeline"],
        param_grid=spec["param_grid"],
        scoring="f1_macro",
        cv=inner_cv,
        n_jobs=1,
        error_score="raise",
        refit=True
    )

    search.fit(X, y, groups=groups)

    tuned_results[name] = {
        "Best CV Macro F1": search.best_score_,
        "Best Params": search.best_params_
    }

    best_estimators[name] = search.best_estimator_

    joblib.dump({
        "best_estimator": search.best_estimator_,
        "tuned_result": tuned_results[name]
    }, cache_path)

    print("Best CV Macro F1:", round(search.best_score_, 3))
    print("Saved cache:", cache_path)

Algorithm to use in the optimization problem. Default is 'lbfgs'.
To choose a solver, you might want to consider the following aspects:

- 'lbfgs' is a good default solver because it works reasonably well for a wide
class of problems.
- For :term:`multiclass` problems (`n_classes >= 3`), all solvers except
'liblinear' minimize the full multinomial loss, 'liblinear' will raise an
error.
- 'newton-cholesky' is a good choice for
`n_samples` >> `n_features * n_classes`, especially with one-hot encoded
categorical features with rare categories. Be aware that the memory usage
of this solver has a quadratic dependency on `n_features * n_classes`
because it explicitly computes the full Hessian matrix.
- For small datasets, 'liblinear' is a good choice, whereas 'sag'
and 'saga' are faster for large ones;
- 'liblinear' can only handle binary classification by default. To apply a
one-versus-rest scheme for the multiclass setting one can wrap it with the
:class:`~sklearn.multiclass.OneVsRestClassifier`.

Interpretation Notes

The final model treats each blood sample as a separate clinical presentation, allowing multiple time points from the same patient to contribute separate observations. Grouped cross-validation was used so all samples from the same patient were kept in the same fold, preventing patient-level leakage.

The modelling workflow compares six distinct modelling paradigms: tree ensembles, gradient boosting, sparse linear modelling, kernel methods, probabilistic classification, and neural networks. Candidate clinical and metabolomic predictors are first defined using dataset structure and domain relevance, while final feature selection occurs inside the machine learning pipeline using tree-based selection within cross-validation.

The best model should be interpreted using AUROC, average precision, macro F1, balanced accuracy, and class-specific recall rather than accuracy alone, because the clinical problem involves imbalance and the deteriorating class is the class of interest.
