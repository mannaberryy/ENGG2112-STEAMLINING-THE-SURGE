Surge triage fails when risk

Surge Environment

Core problem

Current triage reacts after physiological decline is already observable.

A temporal gap exists between early biochemical disruption and visible deterioration — missed by NEWS2 and SOFA.

NEWS2 limitations

Introduction: Project Objectives

A machine learning pipeline for surge triage prioritisation

Converting admission biomarker data into severity and deterioration predictions that support real-time critical care resource allocation.

Model 1 · CBD Dataset

Severity Classification

Classify admitted patients as Severe or Non-Severe from metabolomic biomarker data at admission — identifying high-risk patients where physiological scoring fails under surge.

Severe Non-Severe 43 patients · 587 features
Model 2 · CLD Dataset

Deterioration Prediction

Classify patients as Deteriorating or Stable using longitudinal biomarker profiles — capturing trajectory before physiological decline is clinically observable.

Deteriorating Stable 339 patients · 6 timepoints
Resource Allocation Framework

2×2 Triage Matrix + Surge Simulation

Integrate both model outputs with real-time ICU capacity into a priority matrix. Simulation compares allocation strategies and recommends the mortality-minimising approach.
